# iOS Weather Widgets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a personal-use iOS app (new repo, `~/weather-widgets-ios`) that shows the same four weather metrics as the `dtf-newtab-queue` extension — temperature, precipitation, AQI+PM2.5, UV index — as four independent WidgetKit widgets on the home screen and lock screen.

**Architecture:** A local Swift package `WeatherCore` (pure business logic: Open-Meteo client, App-Group-backed stores, TTL refresh orchestration, tone/category functions — ported from `weatherApi.js`/`weatherStore.js`/`weatherService.js`/`weatherPresentation.js`) plus a `WeatherWidgetKit` product (generic `TimelineProvider`) both linked into an Xcode project (generated by XcodeGen) with three targets: `WeatherWidgetsApp` (SwiftUI settings screen), `WeatherWidgets` (widget extension, 4 `Widget` kinds in one `WidgetBundle`), sharing state through an App Group.

**Tech Stack:** Swift 6.2 (strict concurrency), Swift Testing (`@Test`/`#expect`), SwiftUI, WidgetKit, Foundation `URLSession`/`UserDefaults`, XcodeGen, Personal Team code signing.

**Spec:** `docs/superpowers/specs/2026-07-14-ios-weather-widgets-design.md` (this repo).

## Global Constraints

- Deployment target: iOS 26.0 (current default for new apps). If your installed Xcode reports a different current SDK version, use that number instead in `project.yml` — it is a plain string, not a compiled API, so there is no compatibility risk in changing it.
- `Packages/WeatherCore/Package.swift` targets the safe floor `.iOS(.v17)` (WidgetKit lock-screen families need iOS 16+); the actual app's deployment target is set separately and independently in `project.yml`.
- Swift 6.2 language mode, strict concurrency checking on. No `@unchecked Sendable` except where explicitly called out with justification (test-only in-memory store using `NSLock`).
- Tests use **Swift Testing** (`import Testing`, `@Test`, `#expect`), not XCTest — a deliberate, behavior-equivalent substitution for the spec's generic "unit tests" requirement; Swift Testing is the current recommended framework and has native `async` test support, which this project's async refresh logic needs.
- Personal Team signing (no paid Apple Developer Program). Bundle IDs: `com.undevy.weatherwidgets` (app), `com.undevy.weatherwidgets.widgets` (extension). App Group: `group.com.undevy.weatherwidgets`.
- Single city, no CoreLocation/geolocation permission anywhere.
- 4 widget kinds (`TemperatureWidget`, `PrecipitationWidget`, `AQIWidget`, `UVIndexWidget`), each `StaticConfiguration` (no `AppIntent` configuration), each supporting `.systemSmall`, `.accessoryCircular`, `.accessoryRectangular`, `.accessoryInline`. `AQIWidget`'s inline family shows only the AQI number, no category/PM2.5.
- Cache TTL: 30 minutes. Reload policy is a request to WidgetKit, not a guarantee.
- No App Store/TestFlight distribution, no interactive widget buttons, no multi-city support, no `.systemMedium`/`.systemLarge` home-screen sizes, no in-app dashboard duplicating the widgets.
- Russian-language labels/categories, metric units — same values as `weatherApi.js`/`weatherPresentation.js`/`newtab.js` today.

```swift
import Foundation

// MARK: - Temperature

public enum TemperatureTone: String, Equatable, Sendable {
    case green, yellow, orange
}

public func temperatureTone(todayAt15: Double, yesterdayAt15: Double) -> TemperatureTone {
    if todayAt15 <= 25 { return .green }
    if todayAt15 >= 30 { return .orange }
    if todayAt15 < yesterdayAt15 { return .green }
    if todayAt15 == yesterdayAt15 { return .yellow }
    return .orange
}

// MARK: - Rain

public enum RainTone: Equatable, Sendable {
    case neutral
    case rain(level: Int)
}

public func rainTone(probability: Int) -> RainTone {
    guard probability > 0 else { return .neutral }
    return .rain(level: Int((Double(probability) / 20).rounded(.up)))
}

// MARK: - UV

public enum UVTone: String, Equatable, Sendable {
    case green, orange, red
}

public func uvTone(_ value: Double) -> UVTone {
    if value <= 2 { return .green }
    if value < 6 { return .orange }
    return .red
}

private let uvLevelBands: [(max: Double, label: String)] = [
    (2, "Низкий"),
    (5, "Умеренный"),
    (7, "Высокий"),
    (10, "Очень высокий"),
    (.infinity, "Экстремальный")
]

public func uvIndexLevel(_ value: Double) -> String {
    (uvLevelBands.first { value <= $0.max } ?? uvLevelBands[uvLevelBands.count - 1]).label
}

// MARK: - AQI

public enum AQITone: String, Equatable, Sendable {
    case green, yellow, orange, red, purple, maroon
}

private struct AQIBand {
    let max: Double
    let tone: AQITone
    let label: String
}

private let aqiBands: [AQIBand] = [
    AQIBand(max: 50, tone: .green, label: "Хорошо"),
    AQIBand(max: 100, tone: .yellow, label: "Умеренно"),
    AQIBand(max: 150, tone: .orange, label: "Вредно для чувствительных групп"),
    AQIBand(max: 200, tone: .red, label: "Вредно"),
    AQIBand(max: 300, tone: .purple, label: "Очень вредно"),
    AQIBand(max: .infinity, tone: .maroon, label: "Опасно")
]

private func aqiBand(for value: Double) -> AQIBand {
    aqiBands.first { value <= $0.max } ?? aqiBands[aqiBands.count - 1]
}

public func usAqiTone(_ value: Double) -> AQITone { aqiBand(for: value).tone }
public func usAqiCategory(_ value: Double) -> String { aqiBand(for: value).label }
```

```swift
import Foundation
@testable import WeatherCore

struct FakeHTTPFetcher: HTTPFetching {
    struct StubbedResponse: Sendable {
        let data: Data
        let statusCode: Int
    }

    var responses: [URL: StubbedResponse] = [:]
    var errorToThrow: Error?

    func fetchData(from url: URL) async throws -> (Data, URLResponse) {
        if let errorToThrow {
            throw errorToThrow
        }
        guard let response = responses[url] else {
            throw WeatherAPIError("FakeHTTPFetcher has no response for \(url)")
        }
        let httpResponse = HTTPURLResponse(
            url: url,
            statusCode: response.statusCode,
            httpVersion: nil,
            headerFields: nil
        )!
        return (response.data, httpResponse)
    }
}
```

```swift
public struct OpenMeteoClient: Sendable {
    private let fetcher: HTTPFetching

    public init(fetcher: HTTPFetching = URLSession.shared) {
        self.fetcher = fetcher
    }

    public struct WeatherResult: Equatable, Sendable {
        public let temperature: Double
        public let temperatureTodayAt15: Double
        public let temperatureYesterdayAt15: Double
        public let uvIndex: Double
        public let uvIndexMax: Double
        public let precipitationProbabilityMax: Int
        public let precipitationStartHour: String?
    }

    public static func forecastURL(latitude: Double, longitude: Double) -> URL {
        var components = URLComponents(string: "https://api.open-meteo.com/v1/forecast")!
        components.queryItems = [
            URLQueryItem(name: "latitude", value: String(latitude)),
            URLQueryItem(name: "longitude", value: String(longitude)),
            URLQueryItem(name: "current", value: "temperature_2m,uv_index"),
            URLQueryItem(name: "daily", value: "uv_index_max"),
            URLQueryItem(name: "hourly", value: "temperature_2m,precipitation_probability"),
            URLQueryItem(name: "past_days", value: "1"),
            URLQueryItem(name: "forecast_days", value: "1"),
            URLQueryItem(name: "timezone", value: "auto")
        ]
        return components.url!
    }

    public func fetchWeather(latitude: Double, longitude: Double) async throws -> WeatherResult {
        let url = Self.forecastURL(latitude: latitude, longitude: longitude)
        let (data, response) = try await fetcher.fetchData(from: url)

        guard let http = response as? HTTPURLResponse, (200..<300).contains(http.statusCode) else {
            let status = (response as? HTTPURLResponse)?.statusCode ?? -1
            throw WeatherAPIError("Open-Meteo forecast request failed with status \(status)")
        }

        let body: ForecastResponse
        do {
            body = try JSONDecoder().decode(ForecastResponse.self, from: data)
        } catch {
            throw WeatherAPIError("Open-Meteo forecast response was not valid JSON")
        }

        guard let temperature = body.current?.temperature2m else {
            throw WeatherAPIError("Open-Meteo response is missing current.temperature_2m")
        }
        guard let uvIndex = body.current?.uvIndex else {
            throw WeatherAPIError("Open-Meteo response is missing current.uv_index")
        }
        guard let dailyDates = body.daily?.time, !dailyDates.isEmpty else {
            throw WeatherAPIError("Open-Meteo response is missing daily.time")
        }
        guard let dailyUvIndexMax = body.daily?.uvIndexMax, dailyUvIndexMax.count == dailyDates.count else {
            throw WeatherAPIError("Open-Meteo response has misaligned daily weather data")
        }

        let todayIndex = dailyDates.count - 1
        let today = dailyDates[todayIndex]
        let uvIndexMax = dailyUvIndexMax[todayIndex]

        let summary = try summarizeHourlyForecast(
            today: today,
            time: body.hourly?.time ?? [],
            temperatures: body.hourly?.temperature2m ?? [],
            probabilities: body.hourly?.precipitationProbability ?? []
        )

        return WeatherResult(
            temperature: temperature,
            temperatureTodayAt15: summary.temperatureTodayAt15,
            temperatureYesterdayAt15: summary.temperatureYesterdayAt15,
            uvIndex: uvIndex,
            uvIndexMax: uvIndexMax,
            precipitationProbabilityMax: summary.precipitationProbabilityMax,
            precipitationStartHour: summary.precipitationStartHour
        )
    }

struct HourlySummary: Equatable {
    let temperatureTodayAt15: Double
    let temperatureYesterdayAt15: Double
    let precipitationProbabilityMax: Int
    let precipitationStartHour: String?
}

func previousLocalDate(_ dateString: String) throws -> String {
    let parts = dateString.split(separator: "-")
    guard parts.count == 3,
          let year = Int(parts[0]), let month = Int(parts[1]), let day = Int(parts[2])
    else {
        throw WeatherAPIError("Open-Meteo response has invalid daily.time[0]")
    }

    var calendar = Calendar(identifier: .gregorian)
    calendar.timeZone = TimeZone(identifier: "UTC")!

    var components = DateComponents()
    components.year = year
    components.month = month
    components.day = day

    guard let date = calendar.date(from: components),
          let previousDate = calendar.date(byAdding: .day, value: -1, to: date)
    else {
        throw WeatherAPIError("Open-Meteo response has invalid daily.time[0]")
    }

    let formatter = DateFormatter()
    formatter.dateFormat = "yyyy-MM-dd"
    formatter.timeZone = TimeZone(identifier: "UTC")
    return formatter.string(from: previousDate)
}

func summarizeHourlyForecast(
    today: String,
    time: [String],
    temperatures: [Double],
    probabilities: [Int]
) throws -> HourlySummary {
    let yesterday = try previousLocalDate(today)
    let todayAt15 = "\(today)T15:00"
    let yesterdayAt15 = "\(yesterday)T15:00"

    guard let todayIndex = time.firstIndex(of: todayAt15), todayIndex < temperatures.count else {
        throw WeatherAPIError("Open-Meteo response is missing hourly timestamp \(todayAt15)")
    }
    guard let yesterdayIndex = time.firstIndex(of: yesterdayAt15), yesterdayIndex < temperatures.count else {
        throw WeatherAPIError("Open-Meteo response is missing hourly timestamp \(yesterdayAt15)")
    }

    let currentDayHours = time.enumerated().filter { $0.element.hasPrefix("\(today)T") }
    guard !currentDayHours.isEmpty else {
        throw WeatherAPIError("Open-Meteo response has no hourly data for \(today)")
    }
    for (index, timestamp) in currentDayHours {
        guard index < probabilities.count else {
            throw WeatherAPIError("Open-Meteo response is missing hourly.precipitation_probability for \(timestamp)")
        }
    }

    let precipitationProbabilityMax = currentDayHours.map { probabilities[$0.offset] }.max() ?? 0
    let startEntry = currentDayHours.first { probabilities[$0.offset] >= 30 }
    let precipitationStartHour = startEntry.map { String($0.element.suffix(5)) }

    return HourlySummary(
        temperatureTodayAt15: temperatures[todayIndex],
        temperatureYesterdayAt15: temperatures[yesterdayIndex],
        precipitationProbabilityMax: precipitationProbabilityMax,
        precipitationStartHour: precipitationStartHour
    )
}
```

```swift
public protocol WeatherKeyValueStore: Sendable {
    func data(forKey key: String) -> Data?
    func set(_ data: Data?, forKey key: String)

extension UserDefaults: WeatherKeyValueStore {
```

`UserDefaults` already has a matching `data(forKey:) -> Data?` method, so only `set(_:forKey:)` needs an explicit implementation to satisfy the protocol (its built-in `set(_:forKey:)` takes `Any?`, not `Data?`).

- [ ] **Step 2: Write the in-memory test double**

```swift
import Foundation

/// `@unchecked Sendable` is justified here: the only mutable state is `storage`,
/// and every access is serialized through `lock` (`NSLock`) — the legitimate,
/// provably-thread-safe exception, not a workaround for a compiler error.
final class InMemoryKeyValueStore: WeatherKeyValueStore, @unchecked Sendable {
    private var storage: [String: Data] = [:]
    private let lock = NSLock()

    func data(forKey key: String) -> Data? {
        lock.lock()
        defer { lock.unlock() }
        return storage[key]
    }

    func set(_ data: Data?, forKey key: String) {
        lock.lock()
        defer { lock.unlock() }
        storage[key] = data
    }
}
```

```swift
        guard let data = try? JSONEncoder().encode(cache) else { return }

        store.set(nil, forKey: Self.key)
    }
```

```swift
import Foundation

public struct WeatherRefreshResult: Equatable, Sendable {
    public let cache: WeatherCache
    public let isStale: Bool
}

public struct WeatherRefreshService: Sendable {
    private let client: OpenMeteoClient
    private let cacheStore: WeatherCacheStore
    private let ttl: TimeInterval
    private let now: @Sendable () -> Date

    public init(
        client: OpenMeteoClient,
        cacheStore: WeatherCacheStore,
        ttl: TimeInterval = 30 * 60,
        now: @escaping @Sendable () -> Date = Date.init
    ) {
        self.client = client
        self.cacheStore = cacheStore
        self.ttl = ttl
        self.now = now
    }

    public func refresh(for location: WeatherLocation) async throws -> WeatherRefreshResult {
        let existingCache = cacheStore.load()

        if let cache = existingCache,
           cache.locationName == location.name,
           now().timeIntervalSince(cache.fetchedAt) < ttl {
            return WeatherRefreshResult(cache: cache, isStale: false)
        }

        async let weather = client.fetchWeather(latitude: location.latitude, longitude: location.longitude)
        async let airQuality = client.fetchAirQuality(latitude: location.latitude, longitude: location.longitude)
        let (weatherResult, airQualityResult) = try await (weather, airQuality)

        let freshCache = WeatherCache(
            locationName: location.name,
            fetchedAt: now(),
            temperature: weatherResult.temperature,
            temperatureTodayAt15: weatherResult.temperatureTodayAt15,
            temperatureYesterdayAt15: weatherResult.temperatureYesterdayAt15,
            uvIndex: weatherResult.uvIndex,
            uvIndexMax: weatherResult.uvIndexMax,
            precipitationProbabilityMax: weatherResult.precipitationProbabilityMax,
            precipitationStartHour: weatherResult.precipitationStartHour,
            usAqi: Int(airQualityResult.usAqi.rounded()),
            pm25: airQualityResult.pm25
        )
        cacheStore.save(freshCache)
        return WeatherRefreshResult(cache: freshCache, isStale: false)
    }
}
```

```swift
import WidgetKit
import WeatherCore
import Foundation

/// The three states a widget can render: no city configured yet, a fetch
/// that failed with nothing to fall back on, or actual data (possibly stale,
/// if a fetch failed but a same-city cache still existed).
public enum WeatherWidgetStatus: Sendable {
    case notConfigured
    case error
    case data(WeatherCache, isStale: Bool)
}

/// Conformed to by each widget kind's own `TimelineEntry`, so one generic
/// provider can build any of the four widgets' timelines.
public protocol WeatherWidgetEntry: TimelineEntry {
    init(date: Date, status: WeatherWidgetStatus)
}

public struct WeatherTimelineProvider<Entry: WeatherWidgetEntry>: TimelineProvider, Sendable {
    private let locationStore: WeatherLocationStore
    private let refreshService: WeatherRefreshService
    private let cacheStore: WeatherCacheStore
    private let placeholderCache: WeatherCache

    public init(
        locationStore: WeatherLocationStore,
        refreshService: WeatherRefreshService,
        cacheStore: WeatherCacheStore,
        placeholderCache: WeatherCache
    ) {
        self.locationStore = locationStore
        self.refreshService = refreshService
        self.cacheStore = cacheStore
        self.placeholderCache = placeholderCache
    }

    public func placeholder(in context: Context) -> Entry {
        Entry(date: .now, status: .data(placeholderCache, isStale: false))
    }

    public func getSnapshot(in context: Context, completion: @escaping (Entry) -> Void) {
        // WidgetKit requires this to return quickly, so it only reads the cache — never the network.
        guard let location = locationStore.load() else {
            completion(Entry(date: .now, status: .notConfigured))
            return
        }
        if let cache = cacheStore.load(), cache.locationName == location.name {
            completion(Entry(date: .now, status: .data(cache, isStale: false)))
        } else {
            completion(Entry(date: .now, status: .data(placeholderCache, isStale: false)))
        }
    }

    public func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> Void) {
        guard let location = locationStore.load() else {
            completion(Timeline(entries: [Entry(date: .now, status: .notConfigured)], policy: .never))
            return
        }

        Task {
            do {
                let result = try await refreshService.refresh(for: location)
                let entry = Entry(date: .now, status: .data(result.cache, isStale: result.isStale))
                let nextReload = result.cache.fetchedAt.addingTimeInterval(30 * 60)
                completion(Timeline(entries: [entry], policy: .after(nextReload)))
            } catch {
                let entry = Entry(date: .now, status: .error)
                let retryAt = Date.now.addingTimeInterval(15 * 60)
                completion(Timeline(entries: [entry], policy: .after(retryAt)))
            }
        }
    }
}
```

Note on the `Task { }` in `getTimeline`: `TimelineProvider.getTimeline` is a synchronous, completion-handler-based protocol requirement defined by WidgetKit (not something we control), so bridging into the `async` `refreshService.refresh(for:)` with an unstructured `Task { }` that calls `completion` at the end is the correct, idiomatic pattern here — not a code smell, because the caller (WidgetKit) cannot be made `async`.

**Files:**
- Modify: `WeatherWidgetsApp/ContentView.swift`

**Interfaces:**

```swift
import SwiftUI
import WeatherCore
import WidgetKit

@MainActor
@Observable
final class CitySettingsModel {
    var cityInput: String = ""
    var currentLocation: WeatherLocation?
    var errorMessage: String?
    var isSaving = false

    private let locationStore = WeatherLocationStore(store: UserDefaults(suiteName: "group.com.undevy.weatherwidgets")!)
    private let client = OpenMeteoClient()

    func load() {
        currentLocation = locationStore.load()
        cityInput = currentLocation?.name ?? ""
    }

    func save() async {
        errorMessage = nil
        isSaving = true
        defer { isSaving = false }

        do {
            let result = try await client.geocodeCity(cityInput)
            let location = WeatherLocation(
                name: result.name,
                country: result.country,
                latitude: result.latitude,
                longitude: result.longitude
            )
            locationStore.save(location)
            currentLocation = location
            WidgetCenter.shared.reloadAllTimelines()
        } catch {
            errorMessage = (error as? WeatherAPIError)?.message ?? "Не удалось найти город"
        }
    }
}

struct ContentView: View {
    @State private var model = CitySettingsModel()

    var body: some View {
        Form {
            Section {
                if let location = model.currentLocation {
                    LabeledContent("Текущий город", value: "\(location.name), \(location.country)")
                }
                TextField("Название города", text: $model.cityInput)
                    .autocorrectionDisabled()

                Button("Сохранить", action: saveButtonTapped)
                    .disabled(model.cityInput.trimmingCharacters(in: .whitespaces).isEmpty || model.isSaving)

                if let errorMessage = model.errorMessage {
                    Text(errorMessage).foregroundStyle(.red).font(.caption)
                }
            } footer: {
                Text("Добавьте виджеты «Погода» через долгое нажатие на домашнем экране или экране блокировки — все 4 подтянут этот город.")
            }
        }
        .task {
            model.load()
        }
    }

    private func saveButtonTapped() {
        Task {
            await model.save()
        }
    }
}

#Preview {
    ContentView()
}
```

Note: uses `.task { model.load() }` rather than a `Task { }` inside `onAppear`, per modern SwiftUI guidance (`.task` cancels/re-runs correctly with the view's lifecycle); the save button spawns its own `Task { }` since a `Button` action closure is synchronous and cannot itself be `async` — that is the legitimate, unavoidable case for a bridging `Task { }`, not an anti-pattern.
